----------------------------------------------------------------
SYSTEM PROMPTS
----------------------------------------------------------------

1. WHISPER PROMPT PARAMETER (domain vocabulary hint)
   Passed per job to reduce hallucination on domain-specific content.
   Not a system prompt — it is a short string bias injected into Whisper.

   General (default):
   "Transcription. Clear speech. Accurate punctuation."

   Legal content (if user selects legal domain):
   "Legal document. Court hearing. Petitioner. Respondent. Advocate.
   Affidavit. Deposition. Jurisdiction. High Court. Supreme Court."

   Medical content (if user selects medical domain):
   "Medical transcription. Patient. Diagnosis. Prescription.
   Physician. Dosage. Clinical notes. Referral."

   General business:
   "Business meeting. Board. Agenda. Minutes. Quarterly. Revenue.
   Stakeholder. Management. Strategy."

   Rule: keep prompt under 100 words. Only include vocabulary
   specific to the content — not full sentences.


2. CLAUDE TRANSCRIPT CLEANUP PROMPT (system prompt)
   Used after Whisper outputs raw text. Strict — no rewrites.

   System:
   "You are a transcript editor. Your only job is to fix punctuation,
   remove filler words such as um, uh, you know, and like, and correct
   obvious mishearing errors where the context makes the correct word
   clear. Do not add any content that was not in the original. Do not
   remove any content except filler words. Do not rephrase, summarise,
   or restructure any sentence. Do not add headings, bullet points, or
   formatting. Return only the corrected transcript as plain text and
   nothing else."

   User message format:
   "Clean up the following transcript:

   [RAW TRANSCRIPT TEXT]"


3. CLAUDE TRANSLATION PROMPT (system prompt)
   Used when user requests transcript translation before dubbing.

   System:
   "You are a professional translator. Translate the transcript below
   into [TARGET LANGUAGE]. Preserve the meaning of every sentence
   exactly. Do not add, remove, or summarise any content. Do not add
   headings, notes, or explanations. Return only the translated
   transcript as plain text and nothing else."

   User message format:
   "Translate the following transcript into [TARGET LANGUAGE]:

   [TRANSCRIPT TEXT]"


4. ELEVENLABS DUBBING
   No system prompt required. ElevenLabs handles the full pipeline
   internally — transcription, translation, and voice synthesis are
   all managed by their API. No prompt access on the user side.


HALLUCINATION RISK SUMMARY:

Tool            Hallucinates?   Mitigation
--------------- --------------- ----------------------------------
Whisper         Yes — known     Use prompt parameter with domain
                issue           vocabulary. Trim silence from
                                audio before sending.
Deepgram        No              Pure ASR — mishears, never invents
AssemblyAI      No              Pure ASR — mishears, never invents
Claude cleanup  Yes if loose    Strict system prompt — see above
Claude translate Yes if loose   Strict system prompt — see above
ElevenLabs      No              Specialized pipeline, no LLM risk

----------------------------------------------------------------
BACKGROUND JOB FLOW
----------------------------------------------------------------

1. User uploads file          -> Job created, credits held
2. Worker processes async     -> Whisper -> Claude -> FFmpeg / ElevenLabs
3. Polling shows live status  -> User sees progress on screen
4. Job completes              -> Credits deducted, output file stored
5. Job fails                  -> Credits fully restored, error shown
6. History                    -> All outputs saved, searchable, downloadable
